SQLite Date Storage Guide

Best practices for storing and working with dates in SQLite

Date Storage Options

1. ISO 8601 String Format

Store dates as text in ISO 8601 format: "YYYY-MM-DD HH:MM:SS.SSS"

✓ Human-readable

✓ Easy to debug

✓ Native SQLite date functions work

✗ Slightly larger storage size

CREATE TABLE events ( id INTEGER PRIMARY KEY, name TEXT, event_date TEXT -- Stored as '2023-10-15 14:30:00' );

2. Unix Timestamp (Integer)

Store dates as integers representing seconds since Jan 1, 1970 (Unix epoch)

✓ Compact storage

✓ Easy date arithmetic

✓ Timezone-independent

✗ Not human-readable

✗ Requires conversion

CREATE TABLE events ( id INTEGER PRIMARY KEY, name TEXT, event_date INTEGER -- Stored as 1697373000 );

3. Julian Day Numbers (Real)

Store dates as real numbers representing Julian day numbers

✓ High precision

✓ Native SQLite date functions work

✓ Good for date calculations

✗ Not human-readable

CREATE TABLE events ( id INTEGER PRIMARY KEY, name TEXT, event_date REAL -- Stored as 2460234.10416667 );

Working with Dates in JavaScript

ISO 8601 Strings

// Storing date const now = new Date().toISOString(); // "2023-10-15T14:30:00.000Z" // Retrieving date const date = new Date(isoString);

Unix Timestamps

// Storing date const timestamp = Math.floor(Date.now() / 1000); // 1697373000 // Retrieving date const date = new Date(timestamp * 1000);

SQLite Date Functions

-- ISO 8601 format works with SQLite date functions SELECT date(event_date) as event_day, datetime(event_date, '+1 day') as next_day FROM events; -- Julian day calculations SELECT julianday('now') - julianday(event_date) as days_ago FROM events;

Timezone Considerations

// Store dates in UTC const utcString = new Date().toISOString(); // Convert to local time when displaying const date = new Date(utcString); const localString = date.toLocaleString(); // For user input, convert to UTC before storing const userDate = new Date(userInputString); const utcForStorage = userDate.toISOString();

Recommendation

For most applications, ISO 8601 string format is the best choice because:

  • It's human-readable for debugging
  • It works natively with SQLite's date/time functions
  • It's easy to work with in JavaScript
  • It includes timezone information (UTC)

Use Unix timestamps only if you need to optimize for storage size or perform extensive date arithmetic.

-- Recommended approach CREATE TABLE tasks ( id INTEGER PRIMARY KEY, title TEXT NOT NULL, created_at TEXT DEFAULT (datetime('now')), updated_at TEXT DEFAULT (datetime('now')), due_date TEXT ); -- Insert with automatic timestamps INSERT INTO tasks (title, due_date) VALUES ('Complete project', '2023-12-31 23:59:59'); -- Query using date functions SELECT * FROM tasks WHERE date(due_date) BETWEEN date('now') AND date('now', '+7 days');
